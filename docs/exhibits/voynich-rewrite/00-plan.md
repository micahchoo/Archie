# Voynich Exhibit Rewrite — Plan & Locked Decisions

**Date:** 2026-05-27
**Status:** Wave 1 (Content) dispatched.
**Governing principle:** Content quality is the foundation. The old exhibit was *factually* and *structurally* bad, so any Archie structure layered on it was bad. Sequence is **content → analysis → structure-ideation → build → verify**. No feature is wired until the analysis demands it.

## Why Voynich fits
Beinecke MS 408 has *genuine* sections (herbal, astronomical, balneological, cosmological, pharmaceutical, recipes) and *genuine* competing interpretations (cipher / hoax / natural-language). That is exactly what Archie's **Sections** and **Readings** features exist to carry — when grounded in real material.

## Locked decisions
- **A1 — "demoing" = show off the product.** Archie's features earn their place by making a genuinely good exhibit.
- **A2 — consolidate to a single source of truth.** Kill the hand-maintained `apps/studio/src/voynich.ts` ↔ `apps/viewer/src/voynich.ts` twins; viewer copy is generated by the publish pipeline.
- **A3 — pull in NEW material and reorganize the core content itself.** Subject matter fully in scope.
- **AV object — IN.** Add one deliberate audio/video object so the AV feature (ADR `archie-av.md`, `t=start,end` selectors) is showcased; the manuscript itself is images-only.
- **Versioning — studio-side demo.** Append-only history DAG (ADR-0003) demos in Studio; the static viewer shows only head revisions. Not contorting the viewer for it.

## Features to show off
Deep-zoom objects (OSD) · Layouts Grid + Narrative · **Sections** (ADR-0005) · Notes (WADM, region-anchored) · **Readings** (ADR-0007 — the headline; across ALL objects, not 2 of 5) · Tags (`purpose: tagging`) · Cross-exhibit links (⌘K `archie://` → `bidar`, which is real + published) · Spatiotemporal selectors (ADR-0006) · Publish pipeline · **AV object** (new).

## Material reassessment (answer to "is new material needed?")
**Yes, broadly.** Not a re-wire — a content acquisition:
1. Real Beinecke MS 408 folio images (public domain) replacing the 5 ad-hoc objects, spanning the manuscript's real sections.
2. The new AV asset (public-domain / appropriately-licensed).
3. Sourced scholarship for accurate captions, the three readings, and section prose.
4. Reusable as-is: `bidar` exhibit (cross-link target, already published with stable IDs).

## Five phases
1. **Content** *(Wave 1 — in progress)* — research the real manuscript, select real folios w/ sources, source the AV asset, produce a sourced content spine + material manifest.
2. **Analysis** — interpretive frameworks (cipher/hoax/natural-language) over the same regions; cross-cutting motifs (→ tags); narrative through-line (→ sections).
3. **Ideation** — map the analysis onto Archie structures; output a design doc bridging analysis → build.
4. **Build** — consolidate data model (A2); author in Studio (objects, sections, readings across all folios, tags, cross-links, AV object) through the append-only spine (ADR-0003); wire the Viewer.
5. **Verify** — update `packages/render-core/src/publish/voynich-readings.test.ts` in lockstep; run publish pipeline; happy-dom mount tests (NOT lightpanda — it wedges here); visual sign-off by human.
