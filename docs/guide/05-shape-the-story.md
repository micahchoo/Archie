# Shape the story

Notes annotate; a **narrative** guides. When you want to walk a visitor through
an exhibit in a deliberate order — map, then photo, then recording — you author a
narrative spine.

![The narrative spine editor: ordered sections, each framing a camera on an object](../screenshots/Screenshot%202026-05-26%20at%2017-42-02%20Archie%20Studio.png)

A narrative is a sequence of **sections**. Each section frames a camera on one
object — the same gesture as drawing a note's region, but here it sets *where the
story looks*. Reorder sections to change the reading; switch objects on the rail
to move the camera from one object to the next across the spine. A handy
shortcut: **add a section from a note** seeds a section's framing and prose from a
note you already wrote.

Three distinct tools, kept apart, give an exhibit its depth:

- **Sections** — the ordered spine (how the visitor *moves*).
- **Readings** — competing interpretations of an object (which one they *see*).
- **Tags** — cross-cutting filters (what they can *find*).

You do not need all three. A single annotated map is a complete exhibit. The
spine is there when the material wants to be read in an order.
