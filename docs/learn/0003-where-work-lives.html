Your Projectsaved & yours

Open an Example and you're in a Playground — explore freely, but changes aren't kept. "Keep a copy" turns it into a Project you own.

Under the hood Browser storage is OPFS; "Save to disk" uses a real folder (File System Access on Chromium) or a .archie.zip elsewhere. On disk it's plain JSON plus media — Git-friendly. The same zip is the unit of async collaboration: share a copy, import changes back, merged by version history.
Try it On the Library home, click Save to disk and pick a folder. From then on it autosaves there — reopen anytime with Open a library….
