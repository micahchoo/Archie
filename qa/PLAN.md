# QA Goal Harness — source of truth

The `/goal` loop (see bottom) drives this. The evaluator is a tool-less fast model:
it judges **only what the turn printed**. So completion is a *computed* verdict
(`node qa/gate.mjs`), never prose. This file is the durable spec; `qa/features.jsonl`
is the canonical ledger; `qa/state.json` tracks the loop.

## Canonical store
- **`qa/features.jsonl`** — one feature per line. Canonical. Append/edit here.
- **`qa/features.csv`** — derived, read-only spreadsheet for human reading (the
  prompt's "single canonical spreadsheet" view). Regenerated by `node qa/sync-csv.mjs`
  every turn and on commit; **never hand-edit it** — edits belong in the JSONL.
  `node qa/gate.mjs --csv` prints the same view to stdout; `--check` fails if stale.
- **`qa/state.json`** — `{ turn, cap, discoveryDryStreak, lastFeatureCount }`.

## Feature record schema
```json
{ "id": "studio-annotation-canvas", "name": "…", "harness": "browser",
  "status": "untested", "severity": "none", "defects": 0,
  "story": "As a … I …", "expected": "…", "edges": ["…"], "tests": ["…"],
  "lastTested": "", "notes": "" }
```
- **harness** ∈ `browser` | `desktop` | `network` — the quarantine lane.
- **status** ∈ `untested` | `pass` | `fail` | `flaky` | `waived`.
- **severity** ∈ `none` | `low` | `medium` | `high` | `critical`.

## Phase loop (one slice per turn, in this order)
1. **Discover** — read code, fan out `Explore`. Write candidates to a temp JSONL and
   record them with `node qa/add-features.mjs <file>` — it **dedupes by id** (drops
   features already in the ledger) and updates `discoveryDryStreak` itself. Never
   hand-append to `features.jsonl` during discovery; ids must stay unique or
   `gate.mjs`/`sync-csv.mjs` will throw. An empty pass (`node qa/add-features.mjs`
   with no new finds) bumps the streak toward convergence.
2. **Test-gen** — fill `tests[]`/`edges[]` for `untested` rows.
3. **Execute** — `pnpm exec vitest` per-app (root binary fails rune tests);
   Playwright + axe-core for browser/responsive/a11y. Record `status`/`severity`/`defects`.
4. **Remediate** — smallest safe fix for `fail` rows (`systematic-debugging`).
5. **Regress** — re-run; flip to `pass` only on fresh green.

## Exit predicate (what `gate.mjs` computes)
`GOAL: COMPLETE` ⟺ **all** of:
- `discoveryDryStreak ≥ 2` (feature set converged — the dry-streak fixed point), AND
- every `harness:browser` row is `pass` or `waived`, AND
- zero rows with `status:fail` AND `severity ∈ {high, critical}`.

No turn cap — the loop runs until the predicate above genuinely holds. The
dry-streak clause is what guarantees termination: discovery must go quiet for two
consecutive passes, so an ever-growing feature set can't keep the loop alive forever.

## Quarantine rules (so the loop never stalls on the undrivable)
- **`desktop`** — Tauri-native flows (OS file dialog, native fs). Test the *logic*
  against the fs-seam test backend in vite dev; mark the native-shell residue
  `status:waived` + open a seed. A `tauri build` smoke runs in CI, not per turn.
  Excluded from exit math.
- **`network`** — remote-IIIF. Use **vendored fixtures** (extend
  `apps/viewer/fixtures/voynich.ts`), not live Yale. Run in a retry lane;
  `ECONNREFUSED`/timeout → `status:flaky` (**never `fail`**). Excluded from exit math.
- **Subjective** — UX friction, "confidence score". Operationalize to objective
  proxies the gate can count (axe violations = 0; shadow-walk Sev≥High = 0). The
  irreducible residue routes to human-gate seed **Archie-79be** — do not self-certify.

## Hard constraints (the autonomous loop must honor)
- NEVER modify a shared test fixture to pass a test — create a new one.
- NEVER touch `script-src` in `tauri.conf.json` (PixiJS needs `unsafe-eval`).
- Each turn ends by printing the `node qa/gate.mjs` verdict line **last**.

## The command
```
/goal Drive the QA pipeline in qa/PLAN.md. Each turn: read qa/features.jsonl + qa/state.json; advance ONE slice in phase order (discover → test-gen → execute → remediate → regress); record any discovered features with `node qa/add-features.mjs <candidates.jsonl>` (dedupes by id + updates the dry-streak — never hand-append); bump qa/state.json turn; run `node qa/sync-csv.mjs` to refresh qa/features.csv, then END the turn by running `node qa/gate.mjs` and pasting its verdict line as the LAST line. Only harness:"browser" rows count toward exit — tag Tauri-native flows harness:"desktop" status:"waived" + open a seed; remote-IIIF flows harness:"network" use vendored fixtures and a retry lane where ECONNREFUSED/timeout => status:"flaky" not "fail". vitest per-app (pnpm exec vitest); Playwright+axe for browser/a11y. NEVER edit a shared fixture to pass a test — make a new one. NEVER touch script-src in tauri.conf.json. Route subjective UX/a11y to human-gate seed Archie-79be; don't self-certify it. DONE when `node qa/gate.mjs` prints "GOAL: COMPLETE". No turn cap.
```
